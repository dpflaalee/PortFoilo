### 🏷 프로젝트명: SNS 형식의 Exchange Diary 사이트 📒
- **인원**: 4명  |  **기간**: 2025.04.09 ~ 2025.04.30
- **Github**: [Diary](https://github.com/Lee-jaemyeong/team-repository.git)
- **Demo Site**: [DemoSite](http://3.39.192.82:8080/)
#### 프로젝트 개요
- **목표**: Spring boot와 JPA 기반의 B2C형 웹 서비스 구축 프로젝트
- **컨셉**: 목표 공유가 가능한 교환일기 사이트
- **기술 스택**: Spring Boot 2.7, JAVA 11 , Spring Security 5.0.7, JPA, Thymeleaf 3.0.0 , MySQL 8.0.33 , Lombok 1.18.28 , JQuery 3.7.1
#### 트러블 슈팅
- **문제 발생**
    - 좋아요 데이터를 직접 저장하지 않는 구조로 인해 조인 쿼리 설계 방식을 선택. 당시 조인 쿼리 사용이 미숙했던 탓에 잘못된 구문 작성으로 기능이 정상적으로 작동하지 않음.
- **해결 과정**
    - 전체 ERD 구조를 다시 분석한 뒤, 필요한 데이터 흐름을 정리하고 조인 쿼리를 재설정. 관련 테이블 간 조인 로직을 명확히 정의하여 기능이 정상 작동됨.
- **성과**
    - **SQL 조인에 대한 이해도를 실무적으로 향상시켰고, 이후 프로젝트에서 복합 관계 기반 조회를 보다 자유롭게 구현할 수 있게 됨.**

### 🏷 프로젝트명: 반려동물 소셜 네트워크 서비스 'SSDAM' 🦝
- **인원**: 7명  |  **기간**: 2025.05.12 ~ 2025.06.15
- **Github**: [Ssdam](https://github.com/dpflaalee/sseudamsseudam)
#### 프로젝트 개요
- **목표**: node.js, React를 이용한 SNS 사이트 구축 프로젝트
- **컨셉**: 반려동물을 키우는 사람을 위한 커뮤니티 기반 SNS 플랫폼
- **기술 스택**: node.js 1.0.0, React 18.3.1 , JavaScript , MySQL 3.14.1 , next.js 13.4.13 , Axios 1.9.0 , Antd 4.24.16 , Redux 4.0.5 , Redux-Saga 1.1.3
#### 트러블 슈팅
1. **데이터 흐름 설정 미스**
    - **문제 발생**
        - 비공개 그룹 가입 로직에서 그룹 리더 정보가 제대로 출력되지 않는 오류가 발생.
        - Props 전달 구조 및 변수명이 불분명하여 컴포넌트 간 데이터 흐름이 끊어짐.
    - **해결 과정**
        - 변수명 명확화 및 props 전달 경로 재구성을 통하여 정상 출력 및 상태 관리 개선.
    - **성과**
        - **React 구조 내 데이터 흐름에 대한 이해도가 향상**되어 이후 **컴포넌트 분기 처리에 유연하게 대응**함.
2. **물리 삭제 방식으로 인한 사용자 데이터 유실**
    - **문제 발생**
        - 사용자가 그룹 탈퇴/삭제 시 해당 사용자가 작성한 게시글까지 동시 삭제되는 오류 발생.
        - 그룹 탈퇴 시 관계가 끊기면서 게시글 테이블이 Cascade 방식으로 연결되어 함께 제거됨.
    - **해결 과정**
        - 게시글을 그룹 상태와 분리. isDeleted 상태 추가로 삭제 대신 비노출 처리 방식으로 전환.
    - **성과**
        - **삭제 정책을 물리 삭제에서 논리 삭제로 변경함으로써 사용자 게시글 기록 보존 및 UI 필터링으로 비노출 처리 구현 가능. 실무 관점에서 삭제 정책 설계의 중요성을 체감함.**

### 🏷 프로젝트명: 반려동물 동반 여행경로 추천 사이트 'TripPaw' 🐾
- **인원**: 6명  |  **기간**: 2025.06.19 ~ 2025.07.15
- **Github**: [TripPaw](https://github.com/syeon279/tripPaw)
- **Demo Site**: [DemoSite]()
#### 프로젝트 개요
- **목표**: Spring Boot, MyBatis, React 를 사용한 수익 창출 사이트 구축 프로젝트
- **컨셉**: 반려동물을 키우는 사람을 위한 커뮤니티 기반 SNS 플랫폼
- **기술 스택**: Spring Boot 2.7.14, JAVA 11, MyBatis 2.3.1 , redis 2.7.14, MySQL 8.0, Spring Security 2.7.14 , React 18.20.0 , Next.js 15.3.5
#### 트러블 슈팅
- **문제 발생**
    - 반려동물 여권 도장 정보 조회 시, 여권-리뷰-여행경로간 복잡한 관계로 인해 resultMap에서 데이터를 불러오지 못하는 오류 발생.
    - 도장은 전체 여행 경로(tripPlan) 기반으로 리뷰 작성 후 생성되어야 하는데, 설계 당시 단일 예약(reserv)을 기준으로 Mapper 구조를 작성하여 관계 누락 발생.
- **해결 과정**
    - Mapper 구조를 tripPlan 기준으로 통합 설계하여 리팩토링 진행. 연관 테이블 재정비하여 통합 조회 가능하도록 개선.
- **성과**
    - **복합 데이터 설계와 연관 Mapper 구조에 대한 실무적 이해도 향상. 팀 내 Entity 설계 변경 시 역할 협의 및 데이터 흐름 점검의 중요성을 체감**

- **디자인과 개발의 융합 경험**: 사용자 중심 콘텐츠 설계 경험을 바탕으로 UI와 기능 흐름을 통합적으로 고려하는 능력을 갖추었으며, 개발자로서도 그 강점을 유지하고 있습니다.

- **성장에 대한 끈기**: 부족한 점을 인지하고 학습과 반복 실습을 통해 기술 이해도를 향상시키고, 프로젝트마다 개선을 경험하며 자기 동기를 유지해왔습니다.

## 🎯 개발자로서의 성장 방향 및 입사 후 포부

> 적응과 성과로 연결하는 융합형 개발자

- **단순한 기능 구현을 넘어 실질적인 문제 해결을 지향**하며, 팀과 함께 가치를 만드는 개발을 추구합니다.
- 프론트엔드부터 백엔드까지, **프로젝트마다 새로운 환경에 적응**하며 기능 구현·예외 처리·데이터 설계 등 실질적인 문제를 해결해왔습니다.
- **지속적인 기술 학습과 확장**
  - SQL 쿼리 최적화 → 대용량 데이터 처리 및 분석 역량 고도화
  - 리눅스 기반 배포 → 쉘스크립트와 네트워크 설정 이해도 향상
  - AWS 서비스 활용 → EC2, S3, RDS 실무 경험을 바탕으로 클라우드 아키텍처 설계 역량 확대 중
- 새로운 기술이나 프레임워크도 빠르게 익혀 프로젝트에 기여할 수 있는 **유연한 학습 태도를 유지하고 있습니다.**
- 문서화, 코드 리뷰, 기술 공유 세션을 자발적으로 진행하며, 팀의 성장과 기술 내재화를 돕는 문화를 중요하게 여깁니다.
